Mechanical Design

Our "Temporal Entertainment Module" (TEM) embodies the industrial spirit of the 1800s. The chassis was modeled in OnShape and constructed primarily from 1/4" plywood, laser-cut at Lab64 and assembled with wood glue.

Key Features:

  • Chassis: A custom 50x50x100cm enclosure stained to resemble antique mahogany.
  • Custom Components: Playfield characters, dials, and wire management covers were 3D printed using PLA.
  • Actuation: Two servos control the mechanicsâ€”one for dispensing prize gears and one for the decorative gear mechanism.

Electrical Design

The electrical system bridges the 3.3V logic of the PIC32 with high-power 5V peripherals.

Core Processing & Power

  • Microcontroller: PIC32MX170F256B running at 20MHz.
  • Power & Signal: 74ACT244 buffers are used to shift logic levels for the LED Matrix.

Sensing & Feedback

  • Inputs: TSSP58038 IR receivers detect ball presence, while Piezo elements provide force-sensitive scoring data.
  • Audio: An Adafruit Audio FX Mini Sound Board triggers background music and "Game Over" cues via UART.
  • Haptics: Vibration motors provide tactile feedback when targets are struck.

Software Architecture

The system runs on the Gen2 Events & Services Framework. The architecture is hierarchical, with the PinballService acting as the master controller.

State Machine Logic

The system behavior is defined by five primary states:

  • WELCOME: The idle attraction state. It displays a welcome pattern on the LED matrix.
  • READY: Upon pressing the Start button, the system resets the score and waits for the ball launch.
  • GAME_IN_PLAY: The active state where the 60-second timer runs. It handles scoring from IR/Piezo sensors and manages the 20-second inactivity timer.
  • TIMEOUT: If the player walks away, the game enters a temporary timeout state before resetting.
  • GAME_OVER: Triggered by losing the ball (after the 30s "Ball Save" period expires) or the game timer ending. This state triggers the gear dispenser.

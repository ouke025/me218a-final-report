5. Software Design

The system runs on a PIC32MX170F256B using the Gen2 Events & Services Framework. The architecture is hierarchical, with the `PinballService` acting as the master controller for sub-services like Audio, LED, and Actuation.

[cite_start]

Figure 1: System State Machine controlling the game flow [cite: 258-289].

5.1 State Machine Logic

The system behavior is defined by five primary states:

  • WELCOME: The idle attraction state. [cite_start]It displays a welcome pattern and waits for the START button press[cite: 258, 293].
    • [cite_start]
  • READY: Upon initialization, the system resets the score and waits for the ball to be launched (IR_LAUNCH)[cite: 260, 297].
  • GAME_IN_PLAY: The active state where the 60-second timer runs. It handles:
      [cite_start]
    • Scoring: Inputs from IR_SCORE and Piezo sensors add points and trigger audio/vibration feedback[cite: 265, 269].
      • [cite_start]
    • Inactivity: If 20 seconds pass without input, the system transitions to TIMEOUT[cite: 328, 330].
    • [cite_start]
  • TIMEOUT: A temporary state that plays an audio cue and waits 5 seconds before returning to WELCOME[cite: 286, 331].
  • GAME_OVER: Triggered by IR_LOSS (after safety net) or the 60s timer. [cite_start]It dispenses reward gears and triggers the decorative gear animation[cite: 284, 316, 370].

5.2 Key Services

    [cite_start]
  • IRSensorService: Polls the Launch, Score, and Loss sensors every 10ms, handling debouncing[cite: 257].
    • [cite_start]
  • LEDService: Manages the "Split Mode" display, updating the countdown timer on Module 1 while scrolling score text on Modules 2-4[cite: 361].
    • [cite_start]
  • GearDispensingService: Activates the servo motor to dispense physical gear tokens upon game completion[cite: 367].

4. Electrical Design

The electrical system bridges the 3.3V logic of the PIC32 with high-power 5V peripherals.

[cite_start]

Figure 2: Complete System Schematic [cite: 376-589].

4.1 Core Processing & Power

    [cite_start]
  • Microcontroller: PIC32MX170F256B running at 20MHz[cite: 382].
    • [cite_start]
  • Logic Levels: A 74ACT244 Buffer/Line Driver is used to step up the 3.3V SPI signals (SCK, SS, SDO) to 5V for the LED Matrix[cite: 485, 569].

4.2 Sensor Interface

    [cite_start]
  • IR Sensing: The system uses TSSP58038 IR receiver modules (active low)[cite: 550]. [cite_start]The emitters are driven by IR204A LEDs controlled by N-Channel MOSFETs (Q1-Q3) pulsing at 38kHz[cite: 538, 540].
    • [cite_start]
  • Piezo Inputs: Piezo elements (Y1-Y3) are connected to analog inputs for force-sensitive scoring[cite: 428].

4.3 Actuation & Output

    [cite_start]
  • Audio: An Adafruit Sound Board is triggered directly via the RB7 (Audio) pin[cite: 454, 518].
    • [cite_start]
  • Servos: Two servos control the mechanicsâ€”one for dispensing gears (Pin 14) and one for the decorative gear mechanism (Pin 15)[cite: 440, 450].
    • [cite_start]
  • Vibration: A dedicated vibration motor is driven by a MOSFET on Pin 11 to provide haptic feedback[cite: 449, 557].
